**背景：** `OnPoll` 的 `finally` 中调用 `_pollTimer?.Start()`，如果 `Stop()` 在 `?.` 检查之后、`Start()` 之前 Dispose 了 timer，会抛 `ObjectDisposedException`。

- [ ] **Step 1: 添加 _stopped 标志**

在现有字段声明区域（`_lock` 之后）添加：

```csharp
private volatile bool _stopped;
```

- [ ] **Step 2: 修改 Stop 方法，先设标志**

```csharp
public void Stop()
{
    _stopped = true;
    _pollTimer?.Stop();
    _pollTimer?.Dispose();
    _pollTimer = null;
}
```

- [ ] **Step 3: 修改 OnPoll 的 finally**

```csharp

{
    if (!_stopped)
        _pollTimer?.Start();
```

- [ ] **Step 4: 修改 Start 方法重置标志**

### Task A5: KeyboardHook CallNextHookEx 顺序统一

**背景：** MouseHook 先调 CallNextHookEx 再触发事件，KeyboardHook 反过来。应统一为先释放钩子链。

**Files:**
- Modify: `Services/KeyboardHook.cs`

- [ ] **Step 1: 重构 HookCallback 为 MouseHook 相同的模式**

需要采用 MouseHook 的 try-catch + 返回值保存结构（不能简单交换两行）：

```csharp
private IntPtr HookCallback(int nCode, IntPtr wParam, IntPtr lParam)
{
    if (nCode >= 0 && (wParam.ToInt32() == WM_KEYDOWN || wParam.ToInt32() == WM_SYSKEYDOWN)
        && lParam != IntPtr.Zero)
    {
        // 先读取数据，再释放钩子链（最佳实践，保证读取原始数据）
        var vkCode = Marshal.ReadInt32(lParam);
        var result = CallNextHookEx(_hookId, nCode, wParam, lParam);
        try
        {
            if (vkCode < 255 && !IsModifierKey(vkCode))
            {
                KeyPressed?.Invoke(vkCode);
            }
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"[KeyboardHook] KeyPressed handler exception: {ex.Message}");
        }
        return result;
    }

    return CallNextHookEx(_hookId, nCode, wParam, lParam);
}
```
