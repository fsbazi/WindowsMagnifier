### Task A4: UIA 连续超时降级机制

**背景：** `AutomationElement.FocusedElement` 是跨进程 COM 调用，目标应用无响应时可能阻塞线程池线程。当前 500ms 超时保护能让调用方放弃等待，但底层线程不会被取消，持续快速打字时可能累积大量悬挂线程。三方审查（代码审查 CR-04、QA E1、性能 F1/F2）一致标记为需要修复。

**Files:**
- Modify: `Services/TrackingManager.cs`

- [ ] **Step 1: 添加连续超时计数器字段**

```csharp
private volatile int _consecutiveUiaTimeouts;
private long _uiaDisabledUntilTicks;
private const int MaxConsecutiveTimeouts = 3;
private const long UiaBackoffTicks = 10 * TimeSpan.TicksPerSecond; // 10秒退避
```

- [ ] **Step 2: 在 TryGetCaretViaUIAutomation 开头添加退避检查**

```csharp
// 连续超时后退避，避免线程池饥饿
if (DateTime.UtcNow.Ticks < Interlocked.Read(ref _uiaDisabledUntilTicks))
    return false;
```

- [ ] **Step 3: 在 DebouncedCaretLookup 的超时 catch 中递增计数**

在 `catch (TimeoutException)` 或超时路径中：
```csharp
var timeouts = Interlocked.Increment(ref _consecutiveUiaTimeouts);
if (timeouts >= MaxConsecutiveTimeouts)
{
    Interlocked.Exchange(ref _uiaDisabledUntilTicks,
        DateTime.UtcNow.Ticks + UiaBackoffTicks);
}
```

- [ ] **Step 4: 在成功获取 caret 时重置计数**

```csharp
Interlocked.Exchange(ref _consecutiveUiaTimeouts, 0);
```

- [ ] **Step 5: 编译验证并提交**

```bash
git commit -m "fix(tracking): UIA 连续超时降级机制，3次超时后退避10秒防止线程池饥饿"
```
