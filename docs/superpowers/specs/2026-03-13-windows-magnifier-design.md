# Windows 桌面放大镜应用设计文档

## 概述

开发一款 Windows 桌面放大镜应用，用于无障碍辅助。支持多显示器、多种跟随模式、可调节放大倍数等功能。

## 技术选型

**方案：C# + WPF (.NET 8)**

理由：
- Windows 原生体验，性能最佳
- 多显示器 API 直接支持
- 屏幕截图、透明窗口、置顶等功能实现简单
- 打包后体积小（约 5-10MB）
- 适合无障碍辅助的高稳定性要求

## 核心功能

### 1. 多显示器支持

**核心交互逻辑：**

```
鼠标移动 → 显示器焦点管理器检测位置 → 判断所属显示器
                                              │
                    ┌─────────────────────────┴─────────────────────────┐
                    ▼                                                   ▼
            鼠标在显示器 A                                         鼠标移到显示器 B
                    │                                                   │
        ┌───────────┴───────────┐                           ┌───────────┴───────────┐
        ▼                       ▼                           ▼                       ▼
   显示器 A 窗口            其他窗口                    显示器 B 窗口            其他窗口
   (活动状态)              (非活动状态)                 (活动状态)              (非活动状态)
        │                       │                           │                       │
        ▼                       ▼                           ▼                       ▼
   实时捕获+放大             显示黑色                    实时捕获+放大             显示黑色
   (高帧率更新)            (暂停捕获)                   (高帧率更新)            (暂停捕获)
```

**关键点：**
- 活动显示器：启动屏幕捕获循环，实时渲染放大内容
- 非活动显示器：显示纯黑色背景，暂停屏幕捕获（节省 CPU/GPU 资源）
- 切换延迟：可配置短暂延迟（如 100ms），避免鼠标快速划过时频繁切换

### 2. 放大倍数调节

- 范围：1x - 9x
- 调节方式：
  - 设置窗口滑块控件
  - 快捷键（Ctrl + +/-）
  - 鼠标滚轮（悬停在放大镜上时）

**渲染计算：**
```
屏幕捕获区域 = 当前鼠标位置 ± (窗口尺寸 / 放大倍数 / 2)
```

### 3. 跟随模式（优先级队列 + 最近激活覆盖）

用户可同时开启多种跟随模式，系统按最近激活自动切换：

**状态流转：**

```
                    用户开启跟随鼠标 + 跟随键盘输入
                                │
                                ▼
                    ┌─────────────────────┐
                    │   默认：跟随鼠标指针  │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                                 ▼
      检测到键盘输入                      检测到鼠标移动
              │                                 │
              ▼                                 ▼
    ┌─────────────────┐               ┌─────────────────┐
    │ 跟随键盘输入     │               │ 跟随鼠标指针     │
    │ (优先级高)       │               │ (恢复默认)       │
    └────────┬────────┘               └─────────────────┘
             │
             │ 键盘停止活动 2 秒
             │ (可配置)
             ▼
    ┌─────────────────┐
    │ 恢复跟随鼠标指针 │
    └─────────────────┘
```

**优先级规则（从高到低）：**

| 模式 | 触发条件 | 实现方式 |
|------|----------|----------|
| 跟随鼠标指针 | 鼠标移动事件 | 全局 `MouseHook` |
| 跟随键盘输入 | 键盘按键事件 | 全局 `KeyboardHook` + 获取光标位置 |
| 跟随键盘焦点 | 焦点变化事件 | UI Automation API (`AutomationFocusChangedEvent`) |

### 4. 启动后最小化

- 开启：启动后放大镜窗口最小化，仅显示托盘图标
- 关闭：启动后放大镜窗口正常显示

### 5. 配置持久化

- 存储位置：`%AppData%\Magnifier\config.json`
- 启动时自动加载
- 设置变更即时保存

## 放大镜窗口设计

**窗口特性：**
- 位置：固定在显示器顶部
- 大小：高度可调节（默认 100-200px），宽度等于显示器宽度
- 层级：始终置顶
- 样式：无边框窗口

**行为：**
- 窗口停靠在显示器顶部边缘
- 无法被其他窗口覆盖（`Topmost = true`）
- 不占用任务栏空间
- 支持用户拖拽调整高度

---

## 设置窗口

**配置项：**
- 放大倍数（1-9x）
- 跟随模式（可多选：鼠标/键盘输入/键盘焦点）
- 键盘活动超时（秒）
- 启动后最小化（开/关）
- 放大镜高度（像素）

**配置界面示例：**

```
跟随模式设置：
┌─────────────────────────────────────┐
│ ☑ 跟随鼠标指针                       │

│ ☑ 跟随键盘焦点                       │
│                                     │
│ 键盘活动超时：[  2  ] 秒后恢复鼠标跟随 │
└─────────────────────────────────────┘
```

## 启动与生命周期

**启动流程：**

```
应用启动
    │
    ▼
检测显示器数量 → 为每个显示器创建放大镜窗口
    │
    ▼
加载配置文件 → 应用设置（倍数、高度、跟随模式等）
    │
    ▼
检查"启动后最小化"设置
    │
    ├─ 开启 → 放大镜窗口最小化，仅显示托盘图标
    │
    └─ 关闭 → 放大镜窗口正常显示
```

**关闭行为：**
- 点击关闭按钮 → 最小化到托盘（后台运行）
- 右键托盘图标 → 完全退出

## 技术实现要点

| 功能 | 技术方案 |
|------|----------|
| 屏幕捕获 | `System.Drawing.Graphics.CopyFromScreen` 或 `DXGI Desktop Duplication` |
| 放大渲染 | WPF `WriteableBitmap` + `Image` 控件 |
| 多显示器检测 | `System.Windows.Forms.Screen.AllScreens` |
| 全局鼠标/键盘钩子 | `SetWindowsHookEx` (WH_MOUSE_LL, WH_KEYBOARD_LL) |
| 键盘焦点跟踪 | UI Automation (`Automation.AddAutomationFocusChangedEventHandler`) |
| 窗口置顶 | `Topmost = true` + `WindowStyle = None` |
| 托盘图标 | `System.Windows.Forms.NotifyIcon` |
| 配置存储 | JSON 文件 (`System.Text.Json`) |

## 非功能性需求

- **性能**：屏幕捕获帧率 >= 30 FPS
- **稳定性**：长时间运行无内存泄漏
- **响应性**：鼠标/键盘跟踪延迟 < 50ms
- **资源占用**：CPU < 10%（空闲时），内存 < 100MB
